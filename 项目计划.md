# 销售看板项目开发计划

## 项目概述

前后端分离的销售看板系统，包含：
- **后端** `dashboard/` — Spring Boot 4.0.0 + Java 17
- **前端** `dashboard-web/` — Vue 3 + TypeScript + Vite

---

## 现状

### 前端 (dashboard-web) — 完成度 ~90%
- 页面 UI 全部完成：登录、仪表盘、6个明细分析页、系统管理
- 图表/表格/表单组件封装完整
- 路由、权限、状态管理已配置
- 已有 mock 数据，可直接预览

### 后端 (dashboard) — 完成度 ~5%
- Spring Boot 空骨架，无数据库配置
- 无任何 Entity / Controller / Service
- pom.xml 引用了 MySQL 驱动

### 数据库
- 本地无 MySQL 环境，Docker Desktop 未运行
- 开发阶段使用 **H2 内存数据库** 替代，零配置

## API 路由规划

### 认证
- `POST /api/auth/login` — 登录
- `GET /api/user/info` — 当前用户信息

### 系统管理
- `GET /api/user/list` — 用户列表（分页）
- `POST /api/user` — 新增用户
- `PUT /api/user/{id}` — 编辑用户
- `DELETE /api/user/{id}` — 删除用户
- `GET /api/role/list` — 角色列表（分页）
- `POST /api/role` — 新增角色
- `PUT /api/role/{id}` — 编辑角色
- `DELETE /api/role/{id}` — 删除角色
- `GET /api/menu/list` — 菜单树
- `POST /api/menu` — 新增菜单
- `PUT /api/menu/{id}` — 编辑菜单
- `DELETE /api/menu/{id}` — 删除菜单

### 数据分析
- `GET /api/dashboard/summary` — 仪表盘汇总（6 KPI）
- `GET /api/data/sales` — 销售明细
- `GET /api/data/shipment` — 发货明细
- `GET /api/data/order` — 订单明细
- `GET /api/data/invoice` — 发票明细
- `GET /api/data/deal` — 成交明细
- `GET /api/data/customer` — 客户分析
